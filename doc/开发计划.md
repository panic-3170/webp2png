## 3. 范围界定

### 3.1 In-Scope（本期必做）

- WebP → 透明 PNG 单文件转换工具（PRD §3.1 中 F1–F4、F6）。
- 首页 + 工具页 + 隐私/关于/联系三个静态页。
- SEO 基建：sitemap.xml、robots.txt、JSON-LD、canonical、OG/Twitter Card。
- 暗色模式（跟随系统）。
- 基础分析埋点（Cloudflare Analytics + Plausible）。
- Google AdSense 接入（顶部 728x90、工具完成态下方 336x280 两个核心位）。

### 3.2 Out-of-Scope（本期不做）

- 批量转换（F5，延后到 v1.1）。
- 用户账号 / 历史记录持久化。
- 任何后端文件处理。
- 视频、PDF 等其它工具（按路线图顺次推进）。
- 多语种 i18n。

### 3.3 假设与约束

- 假设 1：单人 / 双人小团队，按“一人前端 + 0.5 人内容/SEO”配置。
- 假设 2：已能注册独立域名并接入 Cloudflare。
- 约束 1：预算 ≤ $10/月（域名 + 可选 Plausible）。
- 约束 2：必须纯前端，禁止后端文件处理。
- 约束 3：浏览器兼容基线 Chrome 90+ / Edge 90+ / Safari 15.4+ / Firefox 90+。

## 5. 技术架构（确认版）

> 详见 PRD §4，本节为落地级补充。

### 5.1 最终选型

| 层      | 选型                                                     |
| ------ | ------------------------------------------------------ |
| 框架     | **Astro 4.x**（SSG、`output: 'static'`）                |
| 样式     | Tailwind CSS 3.x（含 `darkMode: 'media'`）              |
| 核心 JS  | 原生 TS + `OffscreenCanvas` + `createImageBitmap`        |
| 批量打包   | JSZip（v1.1 按需懒加载）                                     |
| 兜底解码   | `@jsquash/webp`（WASM，Feature-detect 后回退）            |
| 托管     | Cloudflare Pages（绑定自定义域、HTTPS、HTTP/3）              |
| 分析     | Cloudflare Web Analytics（无 Cookie） + Plausible（自托管） |
| 广告     | Google AdSense（auto ads 关闭，手动位）                      |
| 表单/反馈  | Formspree 免费档（≤ 50 次/月）                               |
| 代码托管   | GitHub（私有仓库 → Cloudflare Pages 自动化部署）               |

### 5.2 仓库与分支策略

- 主干：`main`（受保护，PR 合入）。
- 开发：`dev`。
- 特性分支：`feat/<scope>`，命名例：`feat/dropzone`、`feat/seo-faq`。
- 发布标签：`v0.1.0-mvp`、`v1.0.0-launch` …

### 5.3 CI/CD

- GitHub Actions：
  - PR 触发：`pnpm install` → `pnpm lint` → `pnpm typecheck` → `pnpm build`。
  - main 合入：自动部署到 Cloudflare Pages（`production`）。
  - dev 分支：自动部署到 `staging` 预览环境。
- 部署密钥走 Cloudflare API Token，存 GitHub Secrets。

### 5.4 关键架构决策（ADR 摘要）

| 决策     | 结论         | 备选         | 理由                  |
| ------ | ---------- | ---------- | ------------------- |
| 框架     | Astro      | Next.js    | 纯静态、SEO 友好、零 hydration 成本 |
| 样式     | Tailwind   | CSS Modules | 工具类样式复用极高            |
| 核心库    | 原生 API     | 引入框架       | 业务简单、SEO 内容为主        |
| 解码     | createImageBitmap 优先 | 直接 WASM    | 兼容性更好、WASM 作为兜底     |
| 监控     | 不用 GA4    | GA4        | 避免影响 AdSense 审核      |

## 11. 质量保证

- **单元测试**：`vitest` 覆盖 `webpToPng.ts` 5 个分支。
- **类型检查**：`tsc --noEmit` 在 CI 必过。
- **Lint**：`eslint` + `prettier`，CI 必过。
- **构建**：`pnpm build` 必过且 `dist/` 大小 < 200KB（不含图片）。

### 11.2 手工

- **Lighthouse**：每次发布前跑一次（mobile + desktop），归档截图。
- **跨浏览器**：Chrome / Edge / Safari / Firefox 桌面 + iOS Safari + Android Chrome。
- **a11y**：键盘 Tab 走查 + VoiceOver 抽样。
- **隐私自检**：DevTools Network 面板确认零上传。
